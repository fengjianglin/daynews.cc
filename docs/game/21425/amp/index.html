一目了然的圖形化指令——《角斗機甲》

天天要聞
2019-12-20

硬核邏輯思維大挑戰

《角斗機甲》是一款在Steam上線的以機器人競技為背景的策略模擬遊戲，玩家在遊戲中需要給己方的機器人設定各種不同的AI策略，然後再與敵方的AI機器人一決勝負。在這個過程中，玩家需要不斷地修改和調整機器人的AI來讓其變得更有策略性更強大，以便戰勝對手完成關卡，這對於玩家的觀察、邏輯能力有不小的挑戰。

機器人大亂斗

《角斗機甲》的框架並不複雜，說到底就是玩家設計一隻機器人隊伍的AI然後與對手去競技決鬥，能夠編輯和設定機器人AI是遊戲本身最大的特色玩法。玩家可以給場上每個機器人單獨設定不同的AI，設定完成後，在機器人相互戰鬥的過程中只能觀察不能干涉。機器人有突擊、散彈、機槍、狙擊四種類型，每種類型機器人的優缺點都非常明顯。比如狙擊手射程遠，但是射速慢、生命值低，適合在保護下遠程狙擊敵人；而機槍手有著最高的血量和護盾，但是它的移動速度極慢無非追擊敵人，只適合當一個固定炮塔。同時遊戲的競技模式也分為收集資源、征服據點、毀滅敵人三種，玩家需要根據不同的模式、地形、敵人來配置自己的機器人AI。

不同類型的機器人性能不同

而遊戲的AI指令非常的基礎，機器人移動、索敵、開火、撤離等策略全都需要玩家設計一步步的詳細指令來完成。比如機器人一個簡單的攻擊敵人命令，就需要給機器人設定動作、目標、範圍等多個參數。而如果是複雜一些的指令，比如「優先擊殺敵人狙擊手」、「敵方散彈手進入中等距離則後退」、「標記和追擊特定目標」等多重條件、行動的命令，則要動用子AI、連接器等規則。

可能許多程序員朋友看到這裡會恍然大悟：這不就是編程嗎？沒錯，《角斗機甲》設定AI的玩法和計算機編程的邏輯基本算是一脈相承：設計AI、測試AI、優化調試、修復BUG，一系列的流程都是程序員非常熟悉的套路。不過在表現形式上，《角斗機甲》當然不會用計算機語言那樣門檻高的方式來為難玩家，將指令圖形化的方法一來便於普通玩家理解，二來也簡化了整個AI編輯的過程。遊戲在初期也提供了提示非常到位的教程，一步步的引導對AI設定編輯不熟悉的玩家。

設計一套自己的AI是遊戲的大部分內容

《角斗機甲》主要內容分別有將近200個關卡的單人戰役模式以及多人線上競技模式兩種，此外還有隨機生產的沙盒模式供玩家自由調試、還原各種情形下的戰鬥。單人戰役中三個模式的關卡都需要逐一挑戰解鎖，每五個關卡和一個BOSS關卡為一章節，完成後會給玩家一定的遊戲幣作為獎勵，而遊戲幣的作用只是用來解鎖機甲的皮膚外觀，完全不影響機甲的性能。遊戲的多人模式採用的並不是即使在線對戰的模式，玩家需要挑戰的是別的玩家已經設置好的機甲AI設定。

金幣只能解鎖一些並不怎麼好看的皮膚

總的來說《角斗機甲》有著非常單純而特別的玩法，玩家在遊戲中只有一個目的：為你的機甲設計好AI。這套玩法在別的遊戲里不少見，《最終幻想12》、《龍騰世紀：審判》中都有設定隊友AI的系統，而單純以「設計AI」為全部內容的遊戲卻並不多了，並且都非常冷門，比如《極客戰記》、《Coding Games》、《Screeps》這類遊戲不要說一般玩家沒有接觸過，它們本身的意義也更多的是用來進行編程教育、提升編程技能的，而不是純粹拿來休閑娛樂的。《角斗機甲》的問題也是如此：對於普通玩家來說，遊戲過於硬核了。

在結束教程之後，遊戲對玩家的指引全部結束，如果不打算去多人模式找虐的話，玩家需要依靠自己去攻克遊戲200多個關卡的單人戰役模式。遊戲本身的機制導致了戰鬥過程中的隨機成分幾乎可以忽略不計，你失敗了就是失敗了，不管重新開始多少次，你的機器人都會以幾乎一樣的方式被敵人按在地上摩擦。於是你不得不進行調整和修改，但調整和修改的過程恰恰是遊戲最艱難的部分。

多人模式的網路問題比較嚴重

首先你要做的就是仔細分析失敗的原因，比如哪個機器人走位失敗導致被擊殺、哪個機器人攻擊方式出錯一直在進行無效攻擊動作、對方哪個機器人需要特別防範應該優先解決等等；然後要做出應對的調整，修改某個機器人的AI，更換隊伍的配置，最後進行測試和校正。但正如剛剛提到的，遊戲在這一過程中不會有任何的提示和指引，玩家的分析、調整、測試每一步都可能犯錯但自己卻渾然不知，最後看著修改過AI後的機器人依然被敵人無情碾壓，可以說是一次很糟糕的反饋。而這種情形只會隨著關卡的進度出現的越來越多。

遊戲的失敗提示是隨機的，沒有實際意義

對於普通玩家來說這種挫敗感是非常致命的，沒有編程基礎的玩家在面對十幾個機器人在場上亂作一團的情形時，連第一步分析錯誤原因都是非常困難的，更別提之後缺乏指引的修改調整要求非常嚴謹的邏輯和記憶能力。面對高難度關卡下多個機器人各不相同的設計需求，普通玩家很可能越改BUG越多、越改越沒有效果，直到最後掀桌崩潰。對於大多數沒有編程基礎的玩家來說，缺乏更進一步的教程以及詳細引導的《角斗機甲》的確過難了。

不過對於程序員來說，《角斗機甲》依然有著不錯的可玩性，雖然有些複雜的邏輯命令在遊戲里無法實現，但通過自己的邏輯能力來戰勝AI的成就感是其它遊戲無法相比的。

「別人設計的AI」，外行人模仿都無從下手

結語

作為一款獨立遊戲，《角斗機甲》有著非常鮮明的特點：它要求玩家有極強的邏輯思維能力。遊戲提供給了玩家豐富的內容、模式來實現自己的聰明才智。但遊戲本身需要一定的編程邏輯思維作為基礎，普通玩家想要體驗遊戲樂趣面臨非常高的門檻，缺乏足夠成熟的指引系統來引導沒有編程基礎的玩家。
