Kafka集群部署

概述

之前的大數據集群主要是離線處理的方式對集群的數據進行開發處理。當前的集群數據量已經達到了PB級別了，離線數據獲取主要是從數倉側進行全量或者增量的方式導入大數據平台，部分是通過SFTP的方式解析進入大數據平台，少量數據是通過介面的方式准實時接入到大數據平台。隨著業務的發展，對於實時數據的接入和應用顯得越來越重要了，接下來的時間會一直更新整個時間數據接入和應用的分享。

我們都是知道kafak是一種分散式的，基於發布/訂閱的消息系統。以時間複雜度為O(1)的方式提供消息持久化能力，即使對TB級以上數據也能保證常數時間複雜度的訪問性能、高吞吐率。這個是非常符合當前接入實時數據進行處理的一個組件。

為什麼說kafka是非常適合我當前的場景呢？當前市場kafka的應用場景常見有：消息系統、跟蹤網站活動、運營指標、日誌聚合、流處理、採集日誌、提交日誌。kafka具有以下的以下優點（特性）：

· 高吞吐、低延遲：kafka 每秒可以處理幾十萬條消息，它的延遲最低只有幾毫秒。

· 可擴展性：kafka 集群支持熱擴展

· 可靠性：消息被持久化到本地磁碟，並且支持數據備份防止數據丟失

· 容錯性：允許集群中節點失敗

· 高並發：支持數千個客戶端同時讀寫

即使在非常廉價的商用機器上也能做到單機支持每秒100K條以上消息的傳輸。支持Kafka Server間的消息分區，及分散式消費，同時保證每個Partition內的消息順序傳輸。同時支持離線數據處理（hive、HBASE）和實時數據處理（spark、storm）。

話不多說，開啟我實際安裝過程。如下的安裝步驟都是可以在虛擬機裡面直接複製使用的。溫馨提醒，如果是在實際的生產環境進行部署的話，請記得掛著磁碟。

一、集群規劃

二、集群環境準備

以下步驟無特殊說明則各節點均需執行

1）關閉防火牆(三台)

# 需要用root用戶執行

service iptables stop

chkconfig iptables off

2）關閉selinux(三台)

vim /etc/selinux/config

# 注釋下面一行

#SELINUX=enforcing

# 添加下面一行

SELINUX=disabled

3）配置域名映射(三台)

vim /etc/hosts

# 在末尾添加以下幾行，注意填寫實際IP及hostname

IP01 hostname01 kafka01

IP02 hostname02 kafka02

IP03 hostname03 kafka03

重啟機器

reboot -h now

4）配置免密登錄

# 生成公鑰與私鑰, 執行以下命令後按三次回車鍵即可

ssh-keygen -t rsa

所有節點拷貝公鑰到kafka01

ssh-copy-id kafka01

複製kafka01認證到其他機器

在kafka01執行

scp /root/.ssh/authorized_keys kafka02:/root/.ssh

scp /root/.ssh/authorized_keys kafka03:/root/.ssh

若中間出錯，100%後面不為1212 (404X3，n個節點為404Xn)，可以通過機器互ping驗證

5）時間同步

在kafka01執行

確認是否安裝ntpd服務

rpm -qa | grep ntp

若未安裝，在線安裝

yum install -y ntp

設置ntpd服務開機啟動

chkconfig ntpd on

修改配置文件

vim /etc/ntp.conf

# 添加下面一行，注意修改子網IP,如192.168.11.0

restrict 子網IP mask 255.255.255.0 nomodify notrap

# 注釋下面四行

#server 0.centos.pool.ntp.org

#server 1.centos.pool.ntp.org

#server 2.centos.pool.ntp.org

#server 3.centos.pool.ntp.org

# 取消注釋或添加下面兩行

server 127.127.1.0 # local clock

fudge 127.127.1.0 stratum 10

配置以下內容，保證BIOS與系統時間同步

vim /etc/sysconfig/ntpd

# 添加下面一行

SYNC_HWLOCK=yes

在其他節點執行

配置另外幾台與kafka01時間同步

crontab -e

*/1 * * * * /usr/sbin/ntpdate kafka01

6）安裝jdk(三台)

查看並卸載自帶的openjdk

rpm -qa | grep java

rpm -e java-1.6.0-openjdk-1.6.0.41-1.13.13.1.el6_8.x86_64 tzdata-java-2016j-1.el6.noarch java-1.7.0-openjdk-1.7.0.131-2.6.9.0.el6_8.x86_64 –kafkaps

創建文件夾

# 用於存放安裝包

mkdir -p /export/software/

# 用於安裝軟體

mkdir -p /export/servers/

上傳到/export/software/並解壓jdk

cd /export/software/

tar -zxvf jdk-8u141-linux-x64.tar.gz -C ../servers/

配置環境變數

vim /etc/profile

# JAVA_HOME

export JAVA_HOME=/export/servers/jdk1.8.0_141

export PATH=:$JAVA_HOME/bin:$PATH

source /etc/profile

驗證

java -version

結果顯示如下

java version “1.8.0_141”

Java(TM) SE Runtime Environment (build 1.8.0_141-b15)

Java HotSpot(TM) 64-Bit Server VM (build 25.141-b15, mixed mode)

三、Zookeeper集群安裝

1）下載安裝包

下載完成後上傳到集群上

2）解壓安裝包

cd /export/software/

tar -zxvf zookeeper-3.4.14.tar.gz -C ../servers/

3）創建軟鏈接

cd /export/servers/

ln -s zookeeper-3.4.14 zookeeper

4）修改配置文件

cd /export/servers/zookeeper/conf

cp zoo_sample.cfg zoo.cfg

mkdir -p /export/servers/zookeeper/zkdatas/

vim zoo.cfg

# 指定zk文件存儲目錄

dataDir=/export/servers/zookeeper/zkdatas

# 取消下面兩行注釋

autopurge.snapRetainCount=3

autopurge.purgeInterval=1

# 添加下面三行

server.1=kafka01:2888:3888

server.2=kafka02:2888:3888

server.3=kafka03:2888:3888

5）添加myid配置

echo 1 > /export/servers/zookeeper/zkdatas/myid

6）分發安裝包

scp -r /export/servers/zookeeper-3.4.14/ kafka02:/export/servers/

scp -r /export/servers/zookeeper-3.4.14/ kafka03:/export/servers/

7）其他機器創建軟鏈接及修改myid值

ln -s /export/servers/zookeeper-3.4.14/ /export/servers/zookeeper

kafka02

echo 2 > /export/servers/zookeeper/zkdatas/myid

kafka03

echo 3 > /export/servers/zookeeper/zkdatas/myid

8）啟動zookeeper集群

啟動腳本如下

zookeeper-start.sh

#!/bin/bash

echo “1.啟動節點kafka01、kafka02和kafka03……”

for n in kafka01 kafka02 kafka03

do

ssh $n “source /etc/profile;/export/servers/zookeeper/bin/zkServer.sh start”

done

#休眠1秒

sleep 1

echo “2.查看集群各節點狀態……”

for n in kafa01 kafka02 kafka03

do

ssh $n “source /etc/profile;/export/servers/zookeeper/bin/zkServer.sh status”

done

停止腳本如下

zookeeper-stop.sh

#!/bin/bash

echo “1.停止節點kafka01、kafka02和kafka03……”

for n in kafka01 kafka02 kafka03

do

ssh $n “source /etc/profile;/export/servers/zookeeper/bin/zkServer.sh stop”

done

#休眠1秒

sleep 1

echo “4. 查看集群各節點狀態……”

for n in kafka01 kafka02 kafka03

do

ssh $n “source /etc/profile;/export/servers/zookeeper/bin/zkServer.sh status”

done

四、kafka集群安裝

1）下載安裝包

下載完成後上傳到集群上

2）解壓安裝包

cd /export/software/

tar -zxvf kafka_2.11-2.2.1.tgz -C ../servers/

3）創建軟鏈接

ln -s kafka_2.11-2.2.1/ kafka

4）創建logs文件夾

cd /export/servers/kafka

mkdir logs

5）備份並修改配置文件

cd /export/servers/kafka/config

cp server.properties server.properties.bak

修改內容

vim server.properties

############################# Server Basics #############################

#broker的全局唯一編號，不能重複

broker.id=0

############################# Socket Server Settings #############################

# 監聽列表

listeners=PLAINTEXT://kafka01:9092

# 處理網路請求的線程數, 一般等於核心數

num.network.threads=3

# 用來處理磁碟IO的線程數

num.io.threads=8

# socket 發送緩衝區

socket.send.buffer.bytes=102400

# socket 接收緩衝區

socket.receive.buffer.bytes=102400

# socket請求最大數值，防止serverOOM

socket.request.max.bytes=104857600

############################# Log Basics #############################

# kafka運行日誌存放的路徑

log.dirs=/export/servers/kafka/logs

# 默認分區數 會被命令行參數覆蓋

num.partitions=3

# 用來恢復和清理data下數據的線程數量

num.recovery.threads.per.data.dir=1

############################# Internal Topic Settings #############################

# 內置主題 “__consumer_offsets” and “__transaction_state” 的副本數

offsets.topic.replication.factor=3

transaction.state.log.replication.factor=3

transaction.state.log.min.isr=3

############################# Log Flush Policy #############################

# The number of messages to accept before forcing a flush of data to disk

#log.flush.interval.messages=10000

# The maximum amount of time a message can sit in a log before we force a flush

#log.flush.interval.ms=1000

############################## Log Retention Policy #############################

# topic數據默認保留時長, 10天

log.retention.hours=240

# A size-based retention policy for logs. Segments are pruned from the log unless the remaining

# segments drop below log.retention.bytes. Functions independently of log.retention.hours.

#log.retention.bytes=1073741824

# 一個消息長度, 超過再創建一個

log.segment.bytes=1073741824

# 文件大小檢查周期

log.retention.check.interval.ms=300000

############################# Zookeeper #############################

# 配置連接Zookeeper集群地址

zookeeper.connect=kafka01:2181,kafka02:2181,kafka03:2181

# zookeeper連接超時時間

zookeeper.connection.timeout.ms=6000

############################# Group Coordinator Settings #############################

group.initial.rebalance.delay.ms=0

############################# Other #############################

# 數據副本個數

default.replication.factor=3

# 單條消息最大長度4Mb

message.max.bytes=4000000

# broker可複製的消息的最大位元組數。值應該比message.max.bytes大

replica.fetch.max.bytes=4194304

# 消費者能讀取的最大消息,值應該大於或等於message.max.bytes

max.partition.fetch.bytes=4194304

# 允許刪除主題

delete.topic.enable=true

6）各節點分別添加環境變數

vim /etc/profile

#KAFKA_HOME

export KAFKA_HOME=/export/servers/kafka/

export PATH=$PATH:$KAFKA_HOME/bin

source /etc/profile

7）分發安裝包

注意：kafka02和kafka03

scp -r /export/servers/kafka_2.11-2.2.1/ kafka02:/export/servers/

scp -r /export/servers/kafka_2.11-2.2.1/ kafka03:/export/servers/

8）其他機器創建軟鏈接及修改配置文件

ln -s /export/servers/kafka_2.11-2.2.1 /export/servers/kafka

kafka02

broker.id=1

listeners=PLAINTEXT://kafka02:9092

kafka03

broker.id=2

listeners=PLAINTEXT://kafka03:9092

9）啟動kafka集群

啟動腳本如下

kafka-start.sh

#!/bin/bash

echo “正在啟動kafka集群……”

for n in kafka01 kafka02 kafka03

do

ssh $n “source /etc/profile;/export/servers/kafka/bin/kafka-server-start.sh -daemon /export/servers/kafka/config/server.properties”

done

echo “啟動kafka集群完成……”

停止腳本如下

kafka-stop.sh

#!/bin/bash

echo “正在關閉kafka集群……”

for n in kafka01 kafka02 kafka03

do

ssh $n “source /etc/profile;/export/servers/kafka/bin/kafka-server-stop.sh”

done

echo “關閉kafka集群完成……”

10）kafka測試

# 創建topic

kafka-topics.sh –create –zookeeper kafka01:2181,kafka02:2181,kafka03:2181 –replication-factor 3 –partitions 3 –topic topic-test

# 查看topic詳情

kafka-topics.sh –describe –zookeeper kafka01:2181,kafka02:2181,kafka03:2181 localhost:2181 –topic topic-test

# 開啟消息生產者

kafka-console-producer.sh –broker-list kafka01:9092,kafka02:9092,kafka03:9092 –topic topic-test

# 開啟消息消費者

kafka-console-consumer.sh –bootstrap-server kafka01:9092,kafka02:9092,kafka03:9092 –from-beginning –topic topic-test

在消息生產者發送消息消費者能收到即可
