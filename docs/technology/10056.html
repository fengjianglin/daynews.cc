在 Linux 上使用 Window Maker 桌面

2019-12-07 141條評論 1,545次閱讀 470人點贊

本文是 24 天 Linux 桌面特別系列的一部分。讓我們和 Window Maker 一起時光倒流，它為如今的用戶實現了老式 Unix NeXTSTEP 環境。– Seth Kenlon（作者）

在 Mac OS X 之前，有一個奇怪的閉源 Unix 系統，稱為 NeXTSTEP 。Sun Microsystems 後來將 NeXTSTEP 的底層設為開放規範，這使其它項目可以創建許多自由開源的 NeXT 庫和組件。GNUStep 實現了許多 NeXTSTEP 庫，而 Window Maker 實現了其桌面環境。

Window Maker 非常接近地模仿了 NeXTSTEP 桌面 GUI，並提供了一個有趣的視角，可以讓人了解 80 年代末 90 年代初的 Unix 是什麼樣子的。它還揭示了窗口管理器（例如 Fluxbox 和 Openbox）背後的一些基本概念。

你可以從發行版的倉庫中安裝 Window Maker。要嘗試它，請在安裝完成後退出桌面會話。默認情況下，會話管理器（KDM、GDM、LightDM 或 XDM，這取決於你的設置）將繼續將登錄到默認桌面，因此登錄時必須覆蓋默認設置。

Selecting the Window Maker desktop in KDM

Window Maker 程序塢

默認情況下，Window Maker 桌面是空的，但每個角落都有幾個程序塢。像在 NeXTSTEP 中一樣，在 Window Maker 中，在程序塢區域，應用可最小化成圖標後停靠，可創建啟動器來快速訪問常見應用，並且可運行微型的 「dockapp」。

你可以在軟體倉庫中搜索 「dockapp」 來試用 dockapp。它們常常是網路和系統監控器、音頻設置面板、時鐘等。這是在 Fedora 上運行的 Window Maker：

Window Maker running on Fedora

應用菜單

要訪問應用菜單，請右鍵單擊桌面上的任意位置。要關閉它，請再次單擊滑鼠右鍵。Window Maker 不是桌面環境（DE），而是一個窗口管理器（DM）。它可以幫助你安排和管理窗口。它唯一捆綁的程序是 WPrefs （或更常見的說法 Window Maker 偏好），它可幫助你配置常用設置，而應用菜單則提供對其他選項（包括主題）的訪問。

運行什麼應用完全由你決定。在 Window Maker 中，你可以選擇運行 KDE 應用、GNOME 應用以及不被視為任何其他主流桌面應用的程序。你可以創建自己的工作環境，並且可以使用 Window Maker 對其進行管理。
