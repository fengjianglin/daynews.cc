我與OpenGL的故事：OpenGL初認識

2019-12-13 694條評論 3,230次閱讀 1224人點贊

OpenGL作為當前主流的圖形API之一，它在一些場合具有比DirectX更優越的特性。

1、OpenGL的優勢

  1. 1、與C語言緊密結合

OpenGL命令最初就是用C語言函數來進行描述的，對於學習過C語言的人來講，OpenGL是容易理解和學習的。如果你曾經接觸過TC的graphics.h，你會發現，使用OpenGL作圖甚至比TC更加簡單。

  1. 2、強大的可移植性

微軟的Direct3D雖然也是十分優秀的圖形API，但它只用於Windows系統。而OpenGL不僅用於Windows，還可以用於Unix/Linux等其它系統，它甚至在大型計算機、各種專業計算機（如：醫療用顯示設備）上都有應用。

並且，OpenGL 的基本命令都做到了硬體無關，甚至是平台無關。

註：我的文章後面會介紹如何用虛擬機安裝Linux（烏班圖），可以關注我啊。

  1. 3、高性能的圖形渲染

OpenGL是一個工業標準，它的技術緊跟時代，現今各個顯卡廠家無一不對OpenGL提供強力支持，激烈的競爭中使得OpenGL性能一直領先。

2、OpenGL的安裝

下面將對Windows下VS2013為例，介紹OpenGL庫的安裝。

OpenGL本質上是一套API圖形介面函數。

GLUT不是OpenGL所必須的，但它會給我們的學習帶來一定的方便，推薦安裝。

Windows環境下的GLUT下載地址：（大小約為150k）

http://www.opengl.org/resources/libraries/glut/glutdlls37beta.zip

無法從以上地址下載的話請使用下面的連接:

http://upload.programfan.com/upfile/200607311626279.zip

Windows環境下安裝GLUT的步驟：

1、將下載的壓縮包解開，將得到5個文件

2、在「我的電腦」中搜索「gl.h」，並找到其所在文件夾（如果是VS2013，則應該是其安裝目錄下面的「VC/PlatformSDK/include/gl文件夾」）。把解壓得到的glut.h放到這個文件夾。

3、把解壓得到的glut.lib和glut32.lib放到靜態函數庫所在文件夾（如果是VS2013，則應該是其安裝目錄下面的「VC/lib」文件夾）。

4、把解壓得到的glut.dll和glut32.dll放到操作系統目錄下面的system32文件夾內。（典型的位置為：C:/Windows/System32）

3、第一個OpenGL程序

  • 3.1 建立項目，插入cpp文件

選擇File->New->Project，然後選擇Win32 Console Application，選擇一個名字，然後按OK。

在談出的對話框左邊點Application Settings，找到Empty project並勾上，選擇Finish。

然後向該工程添加一個代碼文件，取名為「First_OpenGL.c」，注意用.c來作為文件結尾。

選擇控制台應用程序

選擇空項目

在源文件添加新建項(cpp文件)

  • 3.2 寫入代碼

#include <GL/glut.h> //GLUT的頭文件（已經包含<GL/gl.h>和<GL/glu.h>）

void myDisplay(void) // 一個成員函數

{

glClear(GL_COLOR_BUFFER_BIT); // 清除緩衝顏色

//畫矩形，四個參數分別表示了位於對角線上的兩個點的橫、縱坐標

glRectf(-0.5f, -0.5f, 0.5f, 0.5f);

glFlush(); //保證前面的OpenGL命令立即執行（而不是讓它們在緩衝區中等待）

}

int main(int argc, char *argv[])

{

glutInit(&argc, argv); //對GLUT進行初始化，這個函數必須在其它的GLUT使用之前調用一次

glutInitDisplayMode(GLUT_RGB | GLUT_SINGLE); //設置顯示方式

glutInitWindowPosition(100, 100); //設置窗口在屏幕中的位置。

glutInitWindowSize(400, 400);//

glutCreateWindow(“第一個OpenGL程序”);

glutDisplayFunc(&myDisplay);

glutMainLoop();

return 0;

}
