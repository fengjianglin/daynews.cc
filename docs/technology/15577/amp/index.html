嚴格意義上講 Allure 不算是測試框架，但是它是生成漂亮測試報告的開源工具，搭配 Pytest 測試框架食用更搭。

也就是說 Allure 是在 Pytest 執行完生成的測試數據的基礎上，對測試數據進行處理統計，生成格式統一、美觀的測試報告。

一、Allure初識

1. 什麼是 Allure

Allure 是由Qameta Software團隊開源的一款旨在於解決讓每個人能更容易生成並更簡潔閱讀的測試報告框架。它支持大多數的測試框架，如：Pytest、TestNG等，簡單易用便於集成。

二、Pytest集成Allure

Allure要生效需要在測試文件和測試通配文件（conftest.py）中配置 allure。

1. allure的特性

Aullre的幾個特性：

  
    
   
  1. @allure.feature # 用於定義被測試的功能，被測產品的需求點
    2. 
   
  2. @allure.story # 用於定義被測功能的用戶場景，即子功能點
    3. 
   
  3. withallure.step # 用於將一個測試用例，分成幾個步驟在報告中輸出
    4. 
   
  4. allure.attach # 用於向測試報告中輸入一些附加的信息，通常是一些測試數據信息
    5. 
   
  5. @pytest.allure.step # 用於將一些通用的函數作為測試步驟輸出到報告，調用此函數的地方會向報告中輸出步驟
    6.

三、Allure生成測試報告

1. pytest生成測試報告

  
    
   
  1. # 執行全部的測試用例
    2. 
   
  2. pytest ./Allure/test_allure_shoping.py --alluredir ./Allure/result/
    3. 
   
    
   
  3. # 執行指定features或者stories執行一部分測試用例，比如執行『報告購物車』下的『加入購物車』子功能的測試用例
    4. 
   
  4. pytest ./Allure/test_allure_shopping.py --allure_features="報告購物車" --allure_stories="加入購物車" ../Allure/part_result/
    5.

2. Allure生成測試報告

  
    
   
  1. # 生成測試報告
    2. 
   
  2. allure generate ./Alluer/result/ -o ./Allure/report/ --clean
    3. 
   
    
   
  3. # 打開測試報告
    4. 
   
  4. allure open -h 127.0.0.1-p8083./Allure/report/
    5.

打開測試報告，牛皮的是可以切換中文版本
