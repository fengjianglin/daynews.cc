自由軟體之父理查德·斯托曼：被時代拋棄的“偏激者”？

天天要聞
2019-12-20

來源｜極客時間《賣桃者說》

作者｜池建強

編輯｜成敏

你好，這裡是賣桃者說。今天想跟大家聊一聊自由軟體之父理查德·斯托曼的故事，他也是自由軟體運動的精神領袖、GNU 計劃以及自由軟體基金會 (Free Software Foundation) 的創立者。

談到自由軟體，就不得不提它跟開源軟體之間的關係。自由軟體可以說是開源軟體的前身，後者繼承並在一定程度上批判了前者的理念。之後，開源軟體在自由軟體的基礎上蓬勃的發展了起來，直到今天，開源軟體成為軟體開發領域的主流模式之一。

關於這兩者之間的關係，斯托曼自己也曾說過：

「自由軟體」和「開放源碼」或多或少地描述了同一個軟體的類型，但對軟體本身和價值觀的描述是不同的。我們使用「自由軟體」這個術語，來表達自由是重要的思想，而不僅僅是從技術層面出發。

這裡的「同一個軟體的類型」指的是，在發布時公布源代碼，並賦予用戶運行、複製、學習、修改、分發軟體的權利。但相對來講，開源軟體會允許軟體作者保留一定的權利，比如版權、專利權、軟體的開源協議等，而自由軟體的標準更嚴苛，會要求軟體作者開放所有權利，給用戶絕對的自由。

因此，自由軟體更像是一種精神層次上的追求，甚至到後來，很多人提起斯托曼的時候，都脫離了技術身份，轉而用革命家和理論家等詞來形容他。當然，開源軟體的發展之所以能後來居上，跟斯托曼的理論太過極端不無關係。

話題回到斯托曼本身，他和自由軟體之間的故事可以從一台印表機說起。

1980 年，斯托曼所在的 MIT 實驗室購入了一台激光印表機，結果剛用了一段時間印表機就出問題了，很自然的，出於技術人的第一反應，斯托曼想編寫一個提高列印效率的程序，於是跑到印表機廠商哪兒索要印表機程序的源代碼，但卻被告知：「未經授權，不能向他人提供程序源代碼。」這個小小的事件成為了一個導火索。

在此之前，斯托曼和他的同事們組建了一個軟體分享社區，與圈子內外的程序員們分享代碼、交流心得，一起對軟體進行迭代開發。在社區里，成員們都會無償分享自己的代碼，對於軟體，他們提倡的是自由、分享與合作。斯托曼沉迷於這樣的技術氛圍。

然而好景不長，20 世紀 70 年代末，以微軟公司創始人比爾·蓋茨的《致電腦業餘愛好者的一封公開信》為標誌，以世界知識產權組織的《伯爾尼公約》為框架，軟體步入了版權時代。到 80 年代後期，計算機的商業化和軟體專有化席捲了整個產業，一個又一個有才能的 MIT 程序員離開校園，投入市場的懷抱，開始開發專有軟體。

這時候斯托曼感到一個時代結束了。

多年後，斯托曼回憶那段困惑時期，他是這麼說的：

隨著社區的終結，我面臨著一個道德上的抉擇。最簡單的就是投身於專有軟體世界之中，簽署不公開協議，並承諾不幫助同行、同事。自己也很可能編寫軟體，並在不公開協議的前提下發布軟體，去同流合污，迫使更多的人背叛自己的原則。顯然，走這條路，可以掙大錢，而且使編寫代碼的工作增添一份金錢上的快樂。但是我知道，等到自己職業生涯終結時，我再回首這些年為分離人類而砌造的『牆壁』。我會感受到，我將自己的一生都用在使這個世界變得更加糟糕。

於是，斯托曼走上另一條路，他希望建立一套完全自由開放的計算機操作系統，也就是 GNU，讓程序員們可以不被專有軟體限制，可以做自己真正想做的事，也讓社區重新煥發生機。正如他所秉承的理念，「如果你想為這世界做些什麼，僅有理想是不夠的，你需要找一條通往目標的道路並堅持走完。」

由此，斯托曼開始了他的「聖戰」，他的一生都在以「鬥士」的身份對抗著被資本充斥的互聯網行業，並致力於推廣由自由軟體組成的操作系統。斯托曼的對手不是人或公司，而是所有封閉的軟體公司，比如蘋果、微軟、Oracle、IBM 等等。他到目前為止最主要的成就有三個，每一個都是軟體發展史上的光輝：

  • 開發了 Emacs 編輯器。這個不用多做介紹，和 Vim 並稱絕代雙驕，號稱神的編輯器。
  • GNU 通用公共許可證，也就是 GPL，這是世界上採用最為廣泛的自由軟體許可證。
  • Copyleft，所有的 GNU 程序都應遵循「Copyleft」原則，你可以複製、修改、出售自由軟體，但是源代碼相關的改動都必須公開，所有用戶都可以獲得改動後的源碼。Copyleft 保證了自由軟體傳播的延續性。

這些成就保障了自由軟體世界的繁榮昌盛和長治久安，同時催生了一系列優秀的工具軟體和函數庫，比如文本編輯器、編譯器、調試工具、Web 伺服器等等，但 GNU 唯獨沒有開發出操作系統內核，可能是天意使然，就在這個節骨眼上，林納斯開發的 Linux 內核橫空出世了，並最終採用了 GPL 協議，以完全自由、免費的方式發布，從此，GNU 算是功德圓滿了，自由、開源和共享的精神一直延續到了今天。自由軟體服務了千家萬戶，在這樣一個信息共享的時代，我們每個人都或多或少的受益於斯托曼、林納斯和自由軟體的貢獻。

另外，隨著自由軟體的蓬勃發展，GPL 也根據開發者的需求和軟體技術的發展衍生出諸多版本，目前世界上有上百種開源協議，常用的就有 GPL、LGPL、Apache、MIT、BSD 等等，相信軟體開發者都有所了解，這裡就不過多展開了。

斯托曼是一個毫不妥協的理想主義者，此後的 30 余年間，他一直致力於宣傳自由軟體的理念。他開啟的這場自由軟體運動，對於整個軟體行業產生了深遠的影響，甚至可以說，正是這場運動及其後續的衍生，造就了當前軟體開發領域，開源與專有軟體二分天下的現狀。後來人也可以基於他的思想進行繼承和批判，不斷完善開源社會的內在邏輯。

儘管到如今，作為一項運動本身來講，自由軟體已經基本退出了歷史舞台，但斯托曼的思想卻綿延至今，堅持信念、追逐理想的靈魂總是閃閃發光的。
