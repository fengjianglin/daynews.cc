53歲的計算機教授：這些書籍改變了我的編程方式

2019-12-16 3,043條評論 4,814次閱讀 356人點贊

程序員書庫（ID：CodingBook） 猿妹整編

書單來自：https://www.quora.com/As-a-software-engineer-what-are-some-books-that-changed-the-way-you-code/answer/Geraldo-Xex%C3%A9o

作為一名軟體工程師哪些書改變了你的編程方式？也許你還沒有細想過這個問題，不過沒關係，最近，里約熱內盧聯邦大學53歲的GeraldoXexéo教授就在Quora上和大家分享了改變他編程方式的書籍。

GeraldoXexéo是里約熱內盧聯邦大學（UFRJ）的系統工程和計算博士，UFRJ計算機系研究生講師，UFRJ系統工程和計算機專業研究生(碩士和博士)教授，在雜誌和國際會議上發表的幾十篇文章，同時還是多所大學/公司

下面還是來一起看看GeraldoXexéo教授和大家分享的改變他編程方式的書籍吧：

1、《人月神話》

本書內容來自Brooks博士在IBM公司SYSTEM/360家族和OS/360中的項目管理經驗，該項目堪稱軟體開發項目管理的典範。在軟體領域，很少能有像《人月神話》一樣具有深遠影響力和暢銷不衰的著作。

2、《人件》

本書是軟體管理領域的傳奇經典，被譽為「對美國軟體業影響最大的一本書」。全書從管理人力資源、創建健康的辦公環境、僱用並留用正確的人、高效團隊形成、改造企業文化和快樂工作等多個角度闡釋了如何思考和管理軟體開發的最大問題——人（而不是技術），以得到高效的項目和團隊。

3、《代碼大全》

本書從軟體質量和編程思想等方面論述了軟體構造問題，並詳細論述了緊跟潮流的新技術、高屋建瓴的觀點、通用的概念，還含有豐富而典型的程序示例。本書中所論述的技術不僅填補了初級與高級編程技術之間的空白，而且也為程序員們提供了一個有關編程技巧的信息來源。

4、《快速軟體開發》

這本書在總結了包括微軟公司在內的美國軟體業成千上萬個軟體開發項目的實踐經驗、研究成果、經驗教訓的基礎上，詳細列出了幾十種經實踐證明可以直接在軟體開發中應用，以提高開發速度的最佳實踐方法、開發策略、實用技巧等，幫助開發人員和項目經理在了解軟體開發中最常見錯誤的基礎上，根據自身實際情況，制定出滿足項目進度、成本、質量與其他目標要求的最佳方案。

5、《編寫有效用例》

作者Alistair Cockburn，憑藉自己在面向對象領域的豐富經驗，並參考其他專家的建議，擴展了典型的用例處理方法，為軟體開發人員編寫用例提供了一種「基本、具體和實用的」指南。

7、《Scrum實戰》

本書主要是針對敏捷中各種主題有一些闡述，適合打算實現敏捷轉型並導入Scrum的所有人員閱讀，比如開發人員、架構師、測試人員、經理和項目負責人，

8、《C程序設計語言》

這是一本必讀的程序設計語言方面的參考書，人們熟知的「hello,World”程序就是由本書首次引入的，著第2版根據1987年制定的ANSIC標準做了適當的修訂．引入了最新的語言形式，並增加了新的示例，通過簡潔的描述、典型的示例，作者全面、系統、準確地講述了C語言的各個特性以及程序設計的基本方法。

9、《Algorithms + Data Structures = Programs》

它的所有示例都是用現在已經過時的Pascal編程語言編寫的，因此，內容看起來完全過時了，但仍然強烈推薦，書中涵蓋編程基礎(包括遞歸)、許多排序演算法、數據結構以及編譯技術。

10、《資料庫系統導論》

本書全面介紹了現在應用廣泛的資料庫系統，為資料庫技術基礎知識提供堅實的基礎，並對資料庫領域的將來發展方向給出看法，本書一直是資料庫方面的權威著作。本書整體上可以劃分成六個主要部分：基本概念、關係模型、資料庫設計、事務管理、高級專題、對象，關係和XML。第8版已經對資料庫系統目前的系統的最新發展內容進行了擴充；同時又注重於強調概念的理解，而不僅局限於公式的條陳。
