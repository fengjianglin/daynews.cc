7個免費、開源且實用的Node包，你還在等什麼？

俗話說的好：天上不會掉餡餅，天下沒有免費的午餐。

JavaScript圈似乎打破了這一說法，圈友們各個身懷絕技且樂於分享。

眾所周知，JavaScript真正的力量來自於大量可用的開源包。這些免費的開源包促進了JavaScript生態系統的蓬勃發展。我們可以免費使用這些棒呆的包實現自己偉大的編程項目。

今天，小芯將為大家介紹七個超實用的Node包，都是免費的哦！

1. term-size

term-size NPM包能準確確定終端窗口的大小。

term-size 包；來源：Github

當以非交互方式運行時Process.stdout.columns 不存在（例如在子進程中或者管道中）。這個模塊甚至在所有電傳印表機文件描述符被重定向時也能工作。

安裝

$ npm install term-size

用法

const termSize = require(‘term-size’);termSize();

//=> {columns: 143, rows: 24}

Term size 有一個簡短的API

termSize() — 返回一個 具有 columns屬性和 rows屬性的 object（對象）

相關

· term-size-cli — 這個模塊的CLI

注意：確認該包可以在macOS, Linux和 Windows上使用。

2. Node MySQL

Node MySQL是node.js 的一個MySQL驅動程序。用JavaScript編寫，不需要編譯，並且完全由麻省理工學院授權。

Node MySQL.來源: Github

安裝

這是一個Node.js模塊，可通過 npm 註冊表使用 。

安裝之前，下載並安裝Node.js，需要Node.js 0.6或者更高的版本。

使用npm 安裝命令進行安裝：

$ npm install mysql

有關以前0.9.x版本的信息，請訪問 v0.9 分支。

有時可能要求安裝來自Github的最新版本，來檢查bugfix是否有效。這種情況下，請執行以下操作：

$ npm install mysqljs/mysql

建立連接

建立資料庫連接的推薦方式如下：

var mysql = require(‘mysql’);

var connection = mysql.createConnection({

host : ‘example.org’,

user : ‘bob’,

password : ‘secret’

});connection.connect(function(err) {

if (err) {

console.error(‘error connecting: ‘ +err.stack);

return;

} console.log(‘connected as id ‘ + connection.threadId);

});

示例代碼

下面是如何使用該包的示例。

var mysql = require(‘mysql’);

var connection = mysql.createConnection({

host : ‘localhost’,

user : ‘me’,

password : ‘secret’,

database : ‘my_db’

});connection.connect();connection.query(‘SELECT 1 + 1 AS solution’, function(error, results, fields) {

if (error) throw error;

console.log(‘The solution is: ‘,results[0].solution);

});connection.end();

從這個示例中，可以了解：

  • 在連接上調用的每個方法都按順序排隊和執行。
  • 使用 end()關閉連接，確保在quit包發送到伺服器之前執行完所有剩餘的查詢。

3. Commander.js

Commander受Ruby的commander庫啟發，是node.js命令行界面的完整解決方案。

commander.js. 來源：Github

安裝

npm install commander

聲明程序變數

Commander輸出一個全局對象，有助於程序的快速運行。

const program = require(‘commander’);program.version(‘0.0.1’);

對於可能以多種方式使用commander的更大型程序（包括單元測試），最好創建一個本地命令對象來使用。

const commander = require(‘commander’);

const program = new commander.Command();program.version(‘0.0.1’);

命令

可以使用.command為頂級命令指定（子）命令。有兩種實現方法：使用附加在命令上的操作處理程序，或者作為一個獨立的可執行文件（稍後將詳細描述）。在.command的第一個參數中，指定命令名稱和任何命令參數。參數可以是<required>或<optional>，最後一個參數也可能是可變參數……

例如

// Command implemented usingaction handler (description is supplied separately to `.command`)

// Returns new command for configuring.

program

.command(‘clone <source>[destination]’)

.description(‘clone a repository into anewly created directory’)

.action((source, destination) => {

console.log(‘clone command called’);

});// Command implemented usingseparate executable file (description is second parameter to `.command`)

// Returns top-level command for adding more commands.

program

.command(‘start <service>’,’start named service’)

.command(‘stop [service]’, ‘stop namedservice, or all if no name supplied’);

4. Chalk

Chalk是用於終端樣式的Node包。

Chalk. 來源：Github

Chalk有以下亮點：

· 清晰可讀的 API

· 高性能

· 支持嵌套樣式

· 支持256/真彩顏色

· 支持自動檢測顏色

· 不擴展String.prototype

· 清潔和集中

· 積極維護

· 截至2019年10月1日已被大約46,000個包引用

安裝

$ npm install chalk

用法

const chalk = require(‘chalk’);console.log(chalk.blue(‘Hello world!’));

Chalk附帶了一個易於使用的組合API，只需在API中鏈接和嵌套想要的樣式。

const chalk = require(‘chalk’);

const log = console.log;// Combine styled and normal strings

log(chalk.blue(‘Hello’) + ‘ World’ + chalk.red(‘!’));// Compose multiple stylesusing the chainable API

log(chalk.blue.bgRed.bold(‘Hello world!’));// Pass in multiple arguments

log(chalk.blue(‘Hello’, ‘World!’, ‘Foo’, ‘bar’, ‘biz’, ‘baz’));

5. Nodemailer

Nodemailer是Node.js應用程序的一個模塊，可以像發送電子郵件一樣簡單。該項目開始於2010年，那時還沒有發送電子郵件的明智選擇，現在它是大多數Node.js用戶默認的解決方法。

nodemailer. 來源：Github

Nodemailer是麻省理工學院授權註冊的。

準備開始

npm install nodemailer

Nodemailer演示代碼

這是一個完整的示例，用以發送一個有純文本和HTML主體的電子郵件。

const nodemailer =require(‘nodemailer’);// async..await is not allowed in global scope, must usea wrapper

async function main() {

// Generate test SMTP service accountfrom ethereal.email

// Only needed if you don’t have areal mail account for testing

let testAccount = await nodemailer.createTestAccount(); // create reusable transporter object usingthe default SMTP transport

let transporter =nodemailer.createTransport({

host: ‘smtp.ethereal.email’,

port: 587,

secure: false, // true for 465,false for other ports

auth: {

user: testAccount.user, //generated ethereal user

pass: testAccount.pass //generated ethereal password

}

}); // send mail with defined transport object

let info = await transporter.sendMail({

from: ‘”Fred Foo ” <foo@example.com>’, // sender address

to: ‘bar@example.com,baz@example.com’, // list of receivers

subject: ‘Hello ✔’, // Subject line

text: ‘Hello world?’, // plaintext body

html: ‘<b>Helloworld?</b>’ // html body

}); console.log(‘Message sent: %s’, info.messageId);

// Message sent:<b658f8ca-6296-ccf4-8306-87d57a0b4321@example.com> // Preview only available when sendingthrough an Ethereal account

console.log(‘Preview URL: %s’,nodemailer.getTestMessageUrl(info));

// Preview URL:https://ethereal.email/message/WaQKMgKddxQDoou…

}main().catch(console.error);

6. Winston

Winston是一個簡單通用的日誌庫，支持多種傳輸方式。傳輸實質上是日誌的存儲設備。

每個Winston記錄器可以對不同級別配置不同的傳輸方式。例如，用戶可能希望將錯誤日誌儲存在一個永久的遠程位置（如資料庫），但是所有日誌都會輸出到控制台或本地文件。

winston. 來源：Github

Winston旨在將部分日誌記錄進程解耦，增強其靈活性和擴展性。注意支持日誌格式化和級別的靈活性，並確保這些API從運輸日誌（即如何存儲/索引日誌）解耦到公開給程序員。

安裝

npm i Winston

用法

推薦使用的方式是創建自己的日誌記錄器。最簡單的方法是使用winston.createLogger:

const winston = require(‘winston’);const logger =winston.createLogger({

level: ‘info’,

format: winston.format.json(),

defaultMeta: { service: ‘user-service’},

transports: [

//

// – Write to all logs with level`info` and below to `combined.log`

// – Write all logs error (and below)to `error.log`.

//

new winston.transports.File({filename: ‘error.log’, level: ‘error’ }),

new winston.transports.File({filename: ‘combined.log’ })

]

});//

// If we’re not in production then log to the `console` with the format:

// `${info.level}: ${info.message} JSON.stringify({ …rest }) `

//

if (process.env.NODE_ENV !== ‘production’) {

logger.add(newwinston.transports.Console({

format: winston.format.simple()

}));

}

用戶也可以直接通過require(‘winston’)公開的默認日誌記錄器進行記錄，但這僅僅是一種便於共享的日誌記錄器，可以在整個應用程序中使用。

7. Passport

Passport是Node.js的登陸驗證的中間件，可與Express兼容。

Passport的唯一用途是驗證請求，可以通過名為strategies（策略）的可擴展插件集來實現。Passport不掛載路由或承擔任何特定的資料庫模式，極大地提高了靈活性，並允許開發者做出應用程序級的決策。這個API很簡單：提供給Passport一個驗證請求，Passport提供hooks來控制驗證成功或失敗時發生的情況。

passport. 來源：Github

安裝

$ npm install passport

用法策略

Passport使用策略（strategies）的概念去驗證請求。策略包括：通過使用OAuth委託驗證或者使用OpenID進行聯合身份驗證來驗證用戶名和密碼憑據。

在驗證請求之前，必須配置應用程序使用的策略。

passport.use(new LocalStrategy(

function(username, password, done) {

User.findOne({ username: username },function (err, user) {

if (err) { return done(err); }

if (!user) { return done(null,false); }

if (!user.verifyPassword(password)){ return done(null, false); }

return done(null, user);

});

}

));

有480多種策略，可在passportjs.org查找。

會話

Passport包將保持持久的登錄對話。為了持續會話，必須將已驗證身份的用戶序列化到會話，並在發出後續請求時反序列化。

Passport對用戶記錄的存儲方式沒有任何限制。相反，用戶可以為Passport提供函數，使其實現必要的序列化和反序列化邏輯。在典型應用程序中，這和對用戶ID序列化後，再反序列化時通過ID查找用戶一樣簡單。

passport.serializeUser(function(user,done) {

done(null, user.id);

});passport.deserializeUser(function(id, done) {

User.findById(id, function (err, user){

done(err, user);

});

});
