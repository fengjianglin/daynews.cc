輕鬆玩轉java延時隊列DelayQueue

2019-12-15 439條評論 2,368次閱讀 1043人點贊

DelayQueue俗稱延時隊列，他是一個無邊界的BlockingQueue(阻塞隊列)。用於實現了Delayed介面的對象只能在其到期時才能從隊列中取走。這種隊列是有序的，即頭對象的延遲到期的時間最接近當前時間。如果沒有任何延遲到期,那麼就不會有任何頭元素，並且pool將返回null(pool用來檢索並刪除隊列的頭部,如果沒有延遲到期的元素將返回null。所以這也就說明，我們不能將null元素放入該隊列)

有小夥伴們會問什麼時候才會用到延時隊列，目前有這樣一個場景，有10個task執行，但是呢，要求每個任務之間執行的間隔時間必須是2s。這時候是不是用到我們的DelauQueue。

Delayed介面有一個方法名為getDelay()，它可以用來告知延遲到期時間有多長，或者延遲在多長時間之前已經到期，這個方法強制我們使用TimeUnit。

為了排序Delayed介面還繼承了Comparable介面,因此必須實現compareTo()方法,用來編寫自定義的排序機制。

此類的部分方法也會有阻塞，比如take()，當頭部沒有符合要求的對象，則會阻塞等待一個延時到期的元素在此隊列上可用。詳見api

下面是一個簡單的案例,創建不同的DelayedTask任務,且設置超時時間以及排序方式,當在控制台輸出後你可以看到,任務消費的順序都是按照延遲時間的大小排序的。

這裡簡單的介紹一下此案例：

DelayedTask：任務對象實現了Runnable和Delayed介面，實現在延時時間到期後線程在進行執行。

DelayedTask.EndSentinel：內部類，實現收尾工作。

DelayedTaskConsumer：消費延時隊列中的對象

DelayQueueDemo：初始化延時隊列，給每個元素設置延期時間

DelayedTask：

public class DelayedTask implements Runnable, Delayed {
    private static int counter = 0;
    private final int id = counter ++;
    private final int delta;
    private final long trigger;
    protected static List<DelayedTask> sequence = new ArrayList<>();
    public DelayedTask(int delayInMilliseconds) {
        delta = delayInMilliseconds;
        trigger = System.nanoTime() + TimeUnit.NANOSECONDS.convert(delta, TimeUnit.MILLISECONDS);
        sequence.add(this);
    }

    //隊列中按照延遲時間大小排序
    @Override
    public int compareTo(Delayed o) {
        DelayedTask that = (DelayedTask) o;
        if(trigger < that.trigger) return -1;
        if(trigger > that.trigger) return 1;
        return 0;
    }

    //該方法用來告知我們延遲到期時間有多長,或者延遲在多長時間之前已經到期
    //都是以納秒為單位
    @Override
    public long getDelay(TimeUnit unit) {
        return unit.convert(trigger - System.nanoTime(), TimeUnit.NANOSECONDS);
    }

    @Override
    public void run() {
        System.out.println(this + " ");
    }

    @Override
    public String toString() {
        return String.format("[%1$-4d]", delta) + " Task " +id;
    }

    public String summary() {
        return "(" + id + ":" + delta + ")";
    }

    public static class EndSentinel extends DelayedTask{
        private ExecutorService exec;
        public EndSentinel(int delay,ExecutorService e) {
            super(delay);
            exec = e;
        }
        @Override
        public void run() {
            for (DelayedTask delayedTask : sequence) {
                System.out.println(delayedTask.summary() +" ");
            }
            System.out.println(this + "Calling shutdownNow()");
            exec.shutdownNow();
        }

    }
}

DelayedTaskConsumer：

public class DelayedTaskConsumer implements Runnable{

    private DelayQueue<DelayedTask> q;
    public DelayedTaskConsumer(DelayQueue<DelayedTask> q) {
        this.q = q;
    }

    @Override
    public void run() {
        try {
            while(!Thread.interrupted())
                q.take().run();
        } catch (InterruptedException e) {

        }
        System.out.println("Finished DelayedTaskConumer");
    }

}

DelayQueueDemo：

public class DelayQueueDemo {
    public static void main(String[] args) {
        Random rand= new Random(47);
        ExecutorService exec = Executors.newCachedThreadPool();
        DelayQueue<DelayedTask> queue = new DelayQueue<>();
        for(int i = 0; i < 20;i++) {
            queue.put(new DelayedTask(rand.nextInt(5000)));
        }
        queue.add(new DelayedTask.EndSentinel(5000, exec));
        exec.execute(new DelayedTaskConsumer(queue));
    }
}

這個案例有個精妙之處，就是 DelayQueueDemo類最後一個queue.add(new DelayedTask.EndSentinel(5000, exec))，此處向延時隊列中add進行一個內部類元素，該元素將做最後的收尾工作，也可以理解為回收動作，最後將消費者任務關閉掉，因為隊列中並無元素如果不關閉會導致消費者一直處於阻塞狀態。
