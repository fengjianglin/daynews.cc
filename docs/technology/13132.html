瀏覽器語言家族添加一新成員WebAssembly

2019-12-11 3,024條評論 4,835次閱讀 2946人點贊

WebAssembly初出茅廬

萬維網聯盟（W3C）最近宣布，WebAssembly核心規範現已成為正式的Web標準。因此，在HTML，CSS和JavaScript之後，WebAssembly正式成為第四種在瀏覽器中本地運行的語言。

WebAssembly

WebAssembly核心規範將WebAssembly描述為一種安全，可移植的低級代碼語言，目的在於高效執行和緊湊表示。 WebAssembly設計的目的是為了獨立於硬體，獨立於語言和獨立於平台。因此，WebAssembly可以針對所有現代體系結構，台式機或移動設備以及嵌入式系統。 WebAssembly程序可以嵌入在瀏覽器中，作為獨立VM運行或集成在其他環境中。

WebAssembly

為了涵蓋各種用例，將WebAssembly規範拆分並分層為多個文檔。核心規範涉及WebAssembly JS介面，該介面提供了用於與WebAssembly進行交互的JavaScript API；以及WebAssembly Web API，其重點是將WebAssembly與更廣泛的Web平台集成。

WebAssembly的核心是虛擬指令集體系結構（ISA）。核心規範解決了WebAssembly的核心ISA層，定義了指令集，二進位編碼，驗證和執行語義以及文本表示形式。

WebAssembly的潛力：

WebAssembly的到來擴大了僅通過使用Open Web Platform技術應用程序範圍。在機器學習和人工智慧越來越普遍的世界中，不影響用戶安全的情況下在Web上啟用高性能應用程序成為了人們關注的重點。

WebAssembly

我們可以在實用軟體（例如Google Earth埠），標誌性遊戲（例如Doom3）和重型桌面應用程序（例如遊戲編輯器）或設計工具（例如Figma）中觀察到WebAssembly的大量使用。

WebAssembly可以時Web提供強大的計算能力。它將極大地擴展Web應用程序的應用場景。Web技術的實時視頻/音頻處理，硬核遊戲和AI的計算密集型應用程序都成為了它的殺手鐧。

WebAssembly正在成為Web開發人員實施高性能Web Apps的重要選擇。

WebAssembly

WebAssembly的未來版本已經在開發中，其中包括線程，固定寬度SIMD，引用類型，尾部調用或ECMAScript模塊集成之類的功能。

目前瀏覽器中的 MVP（最小化可行產品） 已經很快了。在接下來的幾年裡，隨著瀏覽器的發展和新功能的增加，它將在未來幾年內變得更快。我們已經開始期待新的web應用，期待web4.0。
