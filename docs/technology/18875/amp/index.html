暗網市場再上新貨，下載型木馬病毒“Buer（帕爾）”備受關注

天天要聞
2019-12-16

網路安全公司Proofpoint於近日發文稱，一種名為「Buer（帕爾）」的全新下載型木馬病毒如今已經成長到了足以媲美其老前輩「Smoke Loader」的地步，且已經在暗網市場上積累了一大批忠實粉絲。

暗網廣告及功能概述

據稱，Proofpoint公司最初發現Buer的時間是在今年的8月28日。在當時，它主要被一些攻擊者用來下載並運行Dreambot（銀行木馬Ursnif的一個變種）。

圖1.攜帶Buer的Microsoft Word附件示例

通過在暗網搜索，Proofpoint公司很快發現了有關Buer的銷售廣告。

Buer的定價為400美元，包括配套安裝服務，即你只需要付錢，至於如何將Buer安裝到目標計算機上並確保正常運行全由賣家搞定。

不僅如此，賣家還表示，後續的更新和BIG修復全部免費，但「新建地址」則需要收取25美元的附加費。

圖2. Buer的銷售廣告

廣告同時還列出了Buer的控制面板功能，並指出模塊化Bot是完全採用C語言編寫的，使用的是採用.NET Core編寫的控制面板。

總而言之，賣家想要強調的是，由於編程語言的選擇，無論是Buer的客戶端，還是伺服器，都具有較高的性能。

圖3.Buer控制面板的登錄頁面

根據描述，Buer客戶端的總大小在55至60KB之間，可以作為Windows可執行文件和動態鏈接庫駐留在內存中運行，並且同時兼容32位和64位Windows操作系統。

值得一提的是，該病毒被設定為無法在獨聯體國家（前蘇聯國家，例如俄羅斯）的計算機上運行。

如上所述，由於控制面板是採用.NET Core編寫的，因此能夠輕鬆地在Ubuntu/Debian Linux系統上進行安裝。

控制面板提供了大量的統計信息，包括在線/活躍/離線/總感染主機的數量、受感染主機列表的實時更新、文件下載計數器，同時還支持按操作系統類型、訪問許可權、邏輯CPU內核數對受感染主機進行篩選。

惡意軟體分析

顧名思義，作為一種下載型木馬病毒，Buer能夠下載並執行其他惡意軟體。

反分析功能

Buer具備一些最基本的反分析功能：

  • 通過檢查進程環境塊（PEB）和線程環境塊（TEB）中的NtGlobalFlag來檢查調試器；
  • 使用Red Pill、No Pill以及相關機制來檢查虛擬機；
  • 通過檢查語言代碼，以確保病毒不會在特定國家的計算機上運行。

圖7.硬編碼的語言代碼

長久駐留

Buer能夠通過配置註冊表RunOnce條目來實現在手感染主機上的長久駐留——註冊表項要麼直接執行病毒，要麼安排一個任務來執行它，具體取決於Buer的版本。

命令與控制（C＆C）

命令與控制（C&C）功能通過HTTP(S)的GET請求進行處理，命令信標示例如下圖所示：

圖8. 命令信標示例

這些請求會轉到「update API」，並包含一個加密的參數，該參數可以通過以下方式解密：

  • Base 64解碼；
  • 十六進位解碼；
  • RC4解密（分析樣本中使用的密鑰為「CRYPTO_KEY」）。

以下是明文參數的一個示例：

88a5e68a2047fa5ebdc095a8500d8fae565a6b225ce94956e194b4a0e8a515ae|ab21d61b35a8d1dc4ffb3cc4b75094c31b8c00de3ffaaa17ce1ad15e876dbd1f|Windows 7|x64|4|Admin|RFEZOWGZPBYYOI

它包含使用「|」符號分隔的數據，包括：

  • Bot ID（各種系統參數的SHA-256十六進位摘要，如硬體配置文件GUID和名稱、計算機名稱、卷序列號和CPUID）；
  • 自身可執行映像的SHA-256哈希值；
  • Windows版本；
  • 系統架構；
  • 處理器數量；
  • 用戶許可權；
  • 計算機名稱。

解密後的文本是一個JSON對象，其中包含有關如何下載以及執行有效載荷的各種選項：

type-包含兩種類型：

  • update-更新自身；
  • download_and_exec-下載並執行特定內容。

options-指定要下載的有效載荷的選項：

  • Hash-僅適用於「update」類型，以確認是否存在新的更新；
  • x64-有效載荷是否為64位；
  • FileType-未啟用；
  • AssemblyType-未啟用；
  • AccessToken-用於下載有效載荷；
  • External-指示是從C&C下載，還是從外部URL下載有效載荷。

method-執行Method類：

  • exelocal-創建進程；
  • memload-注入並手動載入有效載荷；
  • memloadex-注入並手動載入有效載荷；
  • loaddllmem-注入並手動載入有效載荷。

Parameters-在命令行中傳遞的參數

pathToDrop-未啟用

autorun-指示是否為有效載荷設置註冊表RunOnce，以實現長久駐留

modules-未啟用

timeout-未啟用

結論

在近期的各種惡意活動中，下載型木馬病毒Buer頻頻出現，作為第二階段有效載荷的惡意軟體包括Dreambot、TrickBot、KPOT、Amadey和Smoke Loader等。

這種全新的下載型木馬病毒具有強大的地理位置定位和反分析功能，且目前正在暗網市場出售。鑒於暗網市場上的廣告以及硬編碼的語言代碼，它的開發者被認為很有可能來自母語是俄語的國家。
