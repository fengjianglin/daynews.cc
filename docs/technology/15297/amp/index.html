你的項目中遇到性能問題了嗎？遇到性能問題你是如何解決的呢？你的解決方式是否正確呢？下面就跟大家一起分享php項目的性能問題。

PHP語言級性能分析

php在什麼情況下會遇到性能問題呢？

在討論性能問題時，我們要明白，整個項目的性能問題不僅僅包含php這一塊，比如還有資料庫服務、緩存服務、文件系統等，因此php性能問題在整個項目中只佔一部分，在優化系統性能時要搞清楚是在優化系統整體性能還是在優化PHP單方面的性能。

首先，PHP是一門編程語言，我們知道每一門編程語言都有自己的優勢及擅長的領域，所以使用php做了它不擅長的事情會很大概率遇到性能問題。再者，編碼者在開發時PHP語法使用不恰當，導致php運行慢。最後，就是我們無法預估的問題，沒有明確解決思路的性能問題。今天我們主要介紹PHP語言級的性能問題。

1、PHP性能檢測的工具：ab

ab(apache benchmark)是由apache提供的壓力測試工具，安裝Apache時會自動安裝此插件。

使用ab工具：

ab -n1000 -c100 https://www.test.com

-n表示請求數，後面跟數字，總共發起的請求數

-c表示並發數，後面跟數字，同一時刻的請求數

url最後跟要檢測的目標地址

我們對百度進行一個檢測，注意：對別的網站進行ab測試時，-n和-c後面的數字小點，不然會對目標網站造成一定的壓力，然後人家發現你就把你給kill了：

ab壓測結果

ab壓測結果

這裡我們只關注兩個數據：

(1)、Requests per second：每秒可處理的請求數，越大越好 。

(2)、Time per request：每個請求處理耗時數，越小越好。

這樣我們在每次進行過性能優化後可以通過前後這兩項數據的對比確定優化有沒有效果。

2、PHP語言級性能優化

(1)、多使用PHP自身攜帶的功能處理邏輯

如：有一個10000個用戶信息的二維數組$userInfoList，要把它轉為一維數組，且以uid為鍵名，username為鍵值，看多很多人寫的代碼都是通過foreach循環來實現，實際上這個是可以用PHP自帶的數組函數來搞定的：

一般代碼：

<?php
$userMap = array();
foreach ($userInfoList as $val) {
 $userMap[$val['uid']] = $val['username'];
}

var_dump($userMap);

php數組函數：

<?php
$userMap = array_column($userInfoList, 'username', 'uid');

var_dump($userMap);

這個函數就能實現這種功能，代碼簡單又容易理解，運行速度還比自己包裝的快。

函數參數：array_column($array, $column, [$index_key]);

(2)、php內置函數的也存在快與慢的差異

比如：查找一個index是否存在於數組$array的key中，大家都能想到使用array_key_exists($index, $array)，當然有的同學也想到了使用isset($array[$index])這個函數。看下面代碼：

<?php
$array = rang(2000, 10000)

$start = time();

for ($i = 0; $i < 20000; $i ++) {

  isset($array[$i]); //測完這個注釋掉，打開下面測

  //array_key_exists($i, $array); //每項要多測幾次，用平均值做比較

}

$end = time();

$time = $end - $start;

echo $time;

沒錯，這兩個函數都能實現相同的功能，然而在經過測算後發現，isset()函數的速度要快於array_key_exists()函數。所以大家平時使用內置函數時，要思考一下什麼時候該使用哪個同類型的函數。

建議phper們多了解些內置函數的實現思路及函數計算的時間複雜度，大概知道了運算的時間複雜度的量級，那麼在使用的時候就能夠選擇合適的函數。

(3)、總結幾個比較常見的注意點

  • 減少PHP魔法函數的使用，也叫魔術方法
  • 最好禁用掉PHP錯誤抑制符@，雖然很方便，但是最好別用
  • 減少在循環體內做大量運算
  • 使用帶引號字元串做鍵值
