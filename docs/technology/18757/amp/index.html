什麼是Three.js？

名字很好理解，Three？三維空間：3D，這樣理解就行，three表3D的意思，js就是JavaScript，連起來就是使用js來寫3D程序，同樣的和JavaScript也是運行在瀏覽器上的。瀏覽器也支持3D程序，它以簡單的、直觀的方式封裝了3D圖形編程中常用的對象，內置了很多對象和極易上手的工具，功能相當強大，確實比CSS3更裝逼，另外它是開源的，你在GitHub上就能找到。

Three.js簡要說明

在WebGl相當出色的一款框架，是一位外國哥們開發的，網名：Mr.doob，框架易用性和擴展性極高，能夠滿足大部分開發者的需求，另外一些功能需二次開發，如：遊戲開發，下面說說一些簡要的功能：

優點：

1、面向對象：Three.js也滿足面向對象，不單單是調用了js函數，也可以直接使用上層的JavaScript對象。

2、包含數學庫：學過數學與計算機科學的人知道，裡面有很多抽象的編程，比如：MATLAB，密碼學等，數學不能太差，裡面有很計算的，Three.js里有很矩陣運算、投影和矢量運算。

3、支持Canvas：CSS3的Canvas也是神一樣的級別，難搞的很，它也支持使用Canvas2D、Canvas3D和Svg進行渲染，因為Three.js兼容性不是很好，有時候還得用他們來解決未兼容的問題。說到這，我說說能兼容的瀏覽器：

主要有：Google Chrome 9+、Firefox 4+、Opera 15+、Safari 5.1+、Internet Explorer 11 和 Microsoft Edge。更低的版本可能要尋求其他的方法了，比如用CAnvas和SVG。

4、擴展性強：你可以按自己的需求來自定義優化或者添加新的特性，亦或者需要某個特殊的數據結構，那麼你只需要封裝到Three.就是即可。

缺點：

沒有學習資料，有也是英文，上網上搜查了以後也是不全的，不適合新手搞，不過現在有了WebGL中文網，還好啦，也不是很全乎，再加上我詳細講解，新手學會運用它絕不是問題。兼容性不太好，但是能兼容目前的主流的瀏覽器。

2、目錄及配置說明

學習之前又不要了解目錄，如下圖所示：

  • .github：在GitHub上的一些說明，如貢獻，發行的版本等
  • build：裡面是Three.js的文件，壓縮和沒壓縮的引用文件
  • docs：這裡是three.js的幫助文檔，裡面是各個函數的api，可惜並沒有詳細的解釋。這也是最煩的，靠這個學習根本整不起來。
  • editor：一個類似3D-max的簡單編輯程序，它能創建一些三維物體。
  • examples：一些很有趣的例子demo，也沒有說明，看代碼蒙蔽，看效果牛逼。
  • src：源代碼目錄，裡面是所有源代碼。
  • files：一些例子的圖片和字體
  • test：一些測試代碼，沒多大用
  • utils：一些腳本，python文件的工具目錄。例如將3D-Max格式的模型轉換為three.js特有的json模型。
  • .editorConfig：幫助開發人員在不同的編輯器和IDE之間定義和維護一致的編碼樣式
  • .gitattribute 指定非文本文件的對比合併方式
  • .gitigonre：用於忽略你不想提交到Git上的文件
  • .npmignore 來忽略你在發布包時不想發布的文件
  • travis.yml 是一種可持續集成(CI)工具，代碼提交後自動構建，與github集成的很好
  • LICENSE：版權信息
  • package.json：主要用來定義項目中需要依賴的包
  • package-lock.json：用以記錄當前狀態下實際安裝的各個npm
  • README.md：介紹three.js的一個文件，裡面還包含了各個版本的更新內容列表。
