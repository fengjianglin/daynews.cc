在這一章里我不打算講太多理論性的東西，甚至在之後的日子裡，我也盡量以輕鬆平常的風格來寫這一個系列的課程，為了讓你們能看懂以及我自己寫著輕鬆。

我最近問了不少朋友學python的目的，有人說為了趕上潮流，為了提高工作效率，甚至小孩子都要學python，不由得讓我想起來之前SOHO老闆的一條微博。

開始我以為是開個玩笑，沒想到現在學這個的人越來越多，python越來越受到關注和喜歡對我來說是一件很高興的事情，畢竟很早以前用python搞開發時老被同行嘲諷：「你們那玩意性能不行，是個玩具」

玩具不玩具暫且不提，既然有人說這是個玩具，那麼我們就用python干一點好玩的事情。

我所理解的python是一門上手比較容易的語言，容易的地方在於它的設計思想或者說作者的思想，用簡單點的話說就是用說人話的方式寫代碼。

怎麼理解呢，我寫代碼舉個例子。

以下是一段go語言在屏幕終端列印helloword的例子

package main

import "fmt"func main() { 
 fmt.Println("hello world")
}

對於初學者來說，恐怕他對於以上代碼會產生各種迷惑，什麼是package main?幹嘛用的， 為main後面這個又是什麼，為什麼不能直接用print列印字元串，要加fmt。諸如此類等等，對於專業的程序員，理解以上代碼毫不費力，但是對於沒有任何編程基礎的人，恐怕還是下面的例子更容易打動人。

以下是pythn的例子

print(“hello world”)

好吧，這個例子我覺得一點都不極端，python基本是就是用說人話的方式去構建代碼，大大解放了人們因為語言特性帶來思想上的束縛，從而把精力花在如何構建一個自己想要的功能實現上，而不是去思考這個語言該怎麼寫，語法是怎麼樣的，到不是說python就沒有語法，而是python到語法已經足夠簡單，簡單到是個正常人稍加培訓就能寫一些代碼的地步，否則也不會有今天這麼多非計算機行業的人士來學它了。

我們接著從剛才那個hello word的例子講起吧。

hello world這個字元串的由來是因為當年貝爾實驗室的一位程序員。於1972年，貝爾實驗室成員布萊恩·柯林漢撰寫的內部技術文件《A Tutorial Introduction to the Language B》首次提到了Hello World這字元串

兩年後，布萊恩·柯林漢和丹尼斯·里奇基於B語言寫成C語言後，在他們撰寫的《C程序設計語言》使用更簡單的方式展示Hello World:

#include <stdio.h>

main()
{
 printf("hello, world\n");

}

自此，Hello World成為了電腦程序員學習新的編程語言的傳統。

由 布萊恩·柯林漢 撰寫的「Hello, world」程序 (1978年)

其實我們通過helloworld這個例子可以了解到 字元串 這個概念，簡單點說就是它可以是中文，英文，數字或者其他你能想到的任意能在屏幕上顯示的內容都可以稱為字元串。

當然我這個話不夠嚴謹，除了字元串，編程語言還有很多別的數據類型。比如整型，浮點數，布爾值等等。但是這不重要，以後我會提到。

現在我們只需要知道，字元串在python當中由引號包裹起來，可以是雙引號，也可以是單引號。

a = ‘我是一個字元串123456’

b = ” 我也是一個字元串12345678″

這裡我定義了兩個變數，分別是a和b，並且給他們賦予了不同的內容，這個過程用計算機的術語來講叫做賦值，分別存儲在內存不同的空間隨時等待著被召喚（例如使用print函數列印它)

以上這樣定義字元串都可以，單引號和雙引號的區別回頭詳細講，反正別用全形的引號即可，您要是來個

a = 「我是字元串」

那計算機就得報錯了

記住這個錯誤提示，字面意思是 語法錯誤，字元驗證失敗，每當出現這種錯誤時，仔細檢查你的字元串里是不是有中文空格，引號，tab鍵之類的。

剛才我們除了用到了print列印一段文字外，還用到了變數定義

就是 a = “我是一個字元串123456”

變數這個概念我想很多朋友應該都有一定的認識，簡單的說是在計算機內存中開闢了一塊空間存儲，哎不對，我說複雜了。

大家暫且理解為變數就是一個代號或者人名吧。比如鋼鐵俠，如果我們每次描述一個人的時候，總要說他是，出生美國幼年喪父母身高170體重70公斤三圍XXX有錢多金帥氣全身盔甲高科技無敵打怪能手拯救地球等等。估計說完他的特點後，就該喝水了，沒別的，太渴。要是老師上課點名來這麼一出估計點完名就該下課了。

於是我們通過人名來定位到某個具體到人，至於這個人的具體特點，那是以後的事情，我們叫人名的那一刻不關心，同樣的，我們在程序里使用變數時也是一樣，定義變數的時候，它的內容是什麼玩意都行，用到的時候再說。

我只需要在用到的時叫它的名字就是了。

同時變數還有一個特點就是它的內容是可以隨時變化的，這玩意人名就做不到了，人可以改名，但是身體內容沒法換。變數可以做到。呃，讓我想想，其實人多身體也可以換，變性嘛。

前天天你是a=”直男”，去了趟東南亞，今天你是a=”少女”

比如我上一刻是

a = “2019快要過去了，我不懷念它”

print(a)

下一刻就是

a = “2020年快要來了，我滿18歲了”

print(a)

那麼之前的那句 “2019快要過去了，我不懷念它”去哪裡了呢？

它被內存丟掉了，內存你們知道吧？所有程序運行時都需要內存，所有數據的呈現都需要在內存中開闢一塊空間(哎，我開始說到這個了吧)，內存的特點就是速度快，用完就丟，像極了渣男。

a這個變數先後被賦值了兩次，第二次的賦值直接將第一次的賦值更改了，第一次的內容也就被內存所丟棄掉了，我們想想，如果它不被丟掉會產生什麼情況？

沒錯，內存被慢慢被這種不釋放的變數撐滿，機器速度變慢。

大家能理解遊戲或者繪圖軟體為什麼耗資源了嗎？因為做圖像處理時會涉及到大量的數據運算，除了CPU之外還需要向內存申請大量的資源。

關於變數的概念這一塊我們暫時講到這裡，變數的命名規則也很簡單，不用能系統關鍵字作為變數名。比如我們開始用到的print，首字母不能是數字，可以是下劃線，python也支持中文變數名，其他語言不一定，不要一概而論。
